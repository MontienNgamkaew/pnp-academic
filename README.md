# 🎓 ระบบบริหารงานวิชาการ (PNP Academic System)
> **ระบบติดตามภารกิจวิชาการและการจัดการส่งแผนการสอนออนไลน์ วิทยาลัยการอาชีพพนมไพร**

ระบบแอปพลิเคชันเว็บระดับพรีเมียม ออกแบบมาเพื่อช่วยอำนวยความสะดวกในการจัดส่ง ตรวจสอบ อนุมัติ และติดตามเอกสารประกอบการจัดการเรียนการสอนของคุณครูผู้สอนได้อย่างครบวงจร ภายใต้หน้าตาผู้ใช้งาน (UI/UX) ที่ทันสมัย สะอาดตา และมีการจัดการข้อมูลอย่างชาญฉลาด

---

## ✨ คุณลักษณะเด่นของระบบ (Key Features)

### 1. 📂 ระบบส่งภารกิจวิชาการหลัก 3 บริการ (Triple Tracking System)
* **โครงการสอน (Syllabus):** ส่งโครงการสอนรายวิชาแบบออนไลน์ พร้อมเงื่อนไขต้องได้รับการอนุมัติจึงจะสามารถพิมพ์บันทึกข้อความราชการได้
* **แผนการจัดการเรียนรู้ (Lesson Plan):** อัปโหลดแผนการจัดการเรียนรู้ประจำรายวิชาแบบรวมไฟล์
* **สื่อการเรียนการสอน (Teaching Materials):** จัดส่งและบันทึกร่องรอยการใช้งานสื่อการสอนย้อนหลัง

### 🎨 2. ระบบปรับแต่งรูปลักษณ์และข้อมูลระบบ (Premium Branding Settings)
* **Dynamic Favicon:** อัปโหลดตราโลโก้ของวิทยาลัยเพื่อนำไปแสดงผลบนแท็บบราวเซอร์ของคุณครูและแอดมินโดยอัตโนมัติ
* **4 Core Themes:** เปลี่ยนโทนสีและการไล่ระดับสีพื้นหลัง (Glow & Gradients) ของทางเข้าหน้าแรกได้ถึง 4 สไตล์:
  * 🌌 **Dark Blue** (น้ำเงินมืดพรีเมียม - ค่าเริ่มต้น)
  * 🔮 **Dark Purple** (ม่วงห้วงอวกาศหรู)
  * 🌲 **Dark Emerald** (เขียวหยกหรูล้ำลึก)
  * 🪨 **Dark Slate** (เทาถ่านหินทันสมัย)
* ปรับแต่งชื่อย่อระบบ (Logo Text) ชื่อระบบย่อ และชื่อวิทยาลัยได้แบบเรียลไทม์ผ่านแดชบอร์ดแอดมิน

### ❄️ 3. แช่แข็งและเก็บประวัติภาคเรียน (Semester Archiving & History View)
* ยกเลิกการลบข้อมูลถาวร (No Truncate) เมื่อขึ้นเทอมใหม่ ระบบจะทำหน้าที่แช่แข็ง (Freeze) ประวัติการส่งและการจัดสอนในเทอมเก่าไว้แบบ **อ่านได้อย่างเดียว (Read-only)**
* เมนู **"ประวัติย้อนหลัง" (History Page)** ให้แอดมินสามารถคลิกเลือกดูรายงานวิเคราะห์ความคืบหน้าของภาคเรียนที่ล่วงเลยไปแล้วย้อนหลังได้ทุกเมื่ออย่างโปร่งใส

### 🗑️ 4. ระบบล้างไฟล์ขยะอัจฉริยะ (Garbage Collection)
* แผงควบคุมล้างขยะอัจฉริยะในหน้าตั้งค่า ช่วยสแกนและเปรียบเทียบไฟล์เอกสารในโฟลเดอร์ `/uploads/` ทั้งหมดกับประวัติในฐานข้อมูล เพื่อคัดและลบไฟล์ขยะที่ซ้ำซ้อนหรือตกค้าง (เช่น ไฟล์ที่ครูอัปโหลดมาใหม่แล้วลบชุดเก่าทิ้ง) พร้อมสรุปพื้นที่คืนให้กับเซิร์ฟเวอร์เป็นหน่วย MB

### 🔒 5. การรักษาความปลอดภัยและประสิทธิภาพการเก็บข้อมูล (Security & Optimization)
* **File Upload Restrictions & 1:1 Course Submissions:** จำกัดรูปแบบไฟล์อัปโหลดเป็นไฟล์ **PDF เท่านั้น** และเพิ่มขนาดสูงสุดเป็น **100 MB** ต่อวิชาต่อประเภทงาน โดยระบบจะเก็บและบำรุงรักษาแบบ 1 วิชาต่อ 1 ไฟล์หลักฐานเพื่อความง่ายในการตรวจสอบ มีระบบตรวจเช็คข้อมูลเก่า ลบไฟล์ขยะสะสมบนดิสก์จริง และอัปเดตเรคคอร์ดเดิมในฐานข้อมูลโดยอัตโนมัติเมื่อครูผู้สอนส่งงานเข้ามาใหม่
* **CSRF Token Protection:** ป้องกันการโจมตีปลอมแปลงคำขอข้ามเว็บไซต์ในจุดทำรายการที่สำคัญทั้งหมด
* **Standard Print Layouts:** หน้าต่างพิมพ์บันทึกข้อความขอส่งโครงการสอนออกแบบระยะขอบกระดาษ A4 ตามระเบียบสารบรรณไทย พร้อมเรียกใช้งานฟอนต์ **TH Sarabun New / Sarabun** อัตโนมัติ

### 👁️ 6. ระบบเปิดดูไฟล์หลักฐานออนไลน์สำหรับแอดมิน (Premium Live Inline PDF Viewer)
* **Split-Screen Design:** หน้าตรวจประเมินผลงานสำหรับผู้ดูแลระบบ (`admin/review.php`) ได้รับการปรับปรุงเป็นโครงสร้างแบบ **Split-Screen (สองคอลัมน์ซ้ายขวา)** เมื่อมีการส่งไฟล์ PDF โดยฝั่งซ้ายจะแสดงแบบฟอร์มประเมินความคิดเห็น และฝั่งขวาจะแสดงหน้าจอตรวจเอกสารหลักฐานผ่าน **Live Inline PDF Viewer** ขนาดใหญ่ที่มีความสูงถึง 650px ช่วยให้แอดมินเปิดอ่านเอกสารได้ทันทีโดยไม่ต้องดาวน์โหลดลงเครื่อง
* **iLovePDF Integration:** แสดงแถบกล่องแนะนำ (Advice Box) ที่ได้รับการดีไซน์สไตล์สีแดงพรีเมียมบนแบบฟอร์มฝั่งครูผู้สอน พร้อมปุ่มเรียกเข้าใช้งาน **iLovePDF Merge PDF** เพื่ออำนวยความสะดวกในการรวมเอกสารการเรียนการสอนหลายแผ่นให้เป็นไฟล์ PDF เพียงไฟล์เดียวก่อนอัปโหลดเข้าสู่ระบบ

## 🛠️ เทคโนโลยีที่เลือกใช้ (Technology Stack)

* **Backend / Core:** PHP 8.x + MySQL / MariaDB (ใช้ PDO ปลอดภัยจาก SQL Injection)
* **Frontend:** HTML5 + JavaScript (ES6) + **Tailwind CSS** (สำหรับระบบ Grid และการแสดงผลที่ตอบสนองบนมือถือ)
* **Typography:** 
  * หน้าหลักเว็บ UI: ฟอนต์ **IBM Plex Sans Thai** (สำหรับภาษาไทย) และ **Outfit** (สำหรับภาษาอังกฤษและตัวเลข)
  * หน้าพิมพ์รายงาน/บันทึกข้อความ: ฟอนต์ **TH Sarabun New / Sarabun** ตามมาตรฐานงานราชการไทย

## 🚀 ขั้นตอนการติดตั้งและทดสอบ (Local Installation)

### 1. การเตรียมสภาพแวดล้อม
* ดาวน์โหลดและติดตั้งโปรแกรมจำลองเซิร์ฟเวอร์ เช่น [XAMPP](https://www.apachefriends.org/)
* นำซอร์สโค้ดนี้ไปวางไว้ในโฟลเดอร์สำหรับรันเว็บ เช่น `C:\xampp\htdocs\pnp-academic`

### 2. การสร้างฐานข้อมูล
1. เปิดเครื่องมือบริหารจัดการฐานข้อมูล (เช่น **phpMyAdmin** หรือโปรแกรมจัดการอื่นๆ)
2. สร้างฐานข้อมูลใหม่ในชื่อ `pnp_academic` พร้อมกำหนด Collation เป็น `utf8mb4_unicode_ci`
3. ทำการนำเข้าโครงสร้างและข้อมูลเริ่มต้นจากไฟล์ SQL ที่เตรียมไว้ในโฟลเดอร์ระบบ:
   `c:\xampp\htdocs\pnp-academic\database\schema.sql`

### 3. การกำหนดค่าการเชื่อมต่อ
* ตรวจสอบความถูกต้องของการเชื่อมต่อฐานข้อมูลได้ที่ไฟล์ `config.php` 
```php
define('DB_HOST', 'localhost');
define('DB_NAME', 'pnp_academic');
define('DB_USER', 'root');
define('DB_PASS', ''); // รหัสผ่านฐานข้อมูล (ค่าเริ่มต้น XAMPP คือว่างเปล่า)
```
